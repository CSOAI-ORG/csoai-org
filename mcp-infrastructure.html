What is the MCP Fleet?

Model Context Protocol (MCP) is the emerging standard for connecting AI systems to external tools, data sources, and services. CSOAI operates a fleet of 202 production-ready MCP servers spanning governance, safety, productivity, research, and infrastructure domains.

Each server is containerised with Docker, health-checked end-to-end, and exposed via streamable HTTP with unified authentication middleware. This is not a prototype — it is live infrastructure.

Technical Architecture

The fleet runs on a unified multi-stage Docker build with Python 3.14, Astral uv, and BuildKit cache mounts. Every service shares:

  • Standardised auth middleware — tiered API key access with Stripe-linked subscription validation
  • Health endpoints — /.well-known/mcp and /health on every container
  • Streamable HTTP transport — via mcp-wrapper.py with consistent routing
  • Fleet-wide test coverage — auth contract, smoke, and endpoint tests distributed across all 202 repos
