The 6 Buses

Six independent verification layers. One trust substrate.

Each bus runs at agent-call latency (<0.1ms). Each is independently verifiable. Each maps to a regulatory requirement.

🪪

Identity Bus — did:csoai

Every agent, user, and system gets a W3C-compliant DID. Verified by the council. Portable across vendors. Sovereign by design.

  • W3C DID v1.1 compliant
  • IETF AIP aligned
  • Self-sovereign key management
  • Portable: works across Vanta, Drata, OneTrust, IBM
🔏

Attestation Bus — Ed25519 + PBFT

Every compliance event becomes a signed receipt with a public verify URL. Your auditor clicks the URL — sees cryptographic proof. Doesn't need to trust CSOAI.

  • Ed25519 signatures (32-byte, 64-char hex)
  • PBFT council consensus (33 nodes, 11 lens types)
  • Public verify URL — no login required
  • Maps to EU AI Act Article 12 (record-keeping) and Article 14 (human oversight)
📜

Policy Bus — PDCA Runtime

Plan-Do-Check-Act enforcement at sub-millisecond latency. Rules run at the agent call. They can't be bypassed by prompt injection or tool injection.

  • Plan: YAML/JSON policy files
  • Do: runtime enforcement on every MCP/A2A call
  • Check: continuous monitoring + drift detection
  • Act: BFT council auto-amends policy on incidents
💳

Payment Bus — Compliance Pre-Check

Every x402, AP2, and ACP payment flow is pre-checked for compliance before the transaction settles. No more paying sanctioned entities by accident.

  • x402 Foundation membership
  • AP2 / ACP integration
  • OpenSanctions real-time screening
  • Sub-millisecond pre-check
📋

Audit Bus — Blockchain Anchored

Every event hash-anchored to Polygon PoA. Tamper-evident. Regulator-checkable. Survives CSOAI, survives any vendor.

  • Polygon PoA anchoring (every 100 events)
  • Ed25519 + optional eIDAS timestamp
  • Regulator dashboard (no CSOAI access required)
  • Public verify URLs for every cert
⚖️

Council Bus — BFT Multi-Stakeholder

33-node BFT council with 11 lens types. 3-provider replicas per lens. Overrides any single-vendor decision. Sovereign governance for the agentic economy.

  • 11 lens types × 3 providers = 33 nodes
  • f = 10 Byzantine tolerance (10 nodes can fail, system survives)
  • Sub-second finality (<5ms)
  • Multi-stakeholder: regulators, industry, civil society
