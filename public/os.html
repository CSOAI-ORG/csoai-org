Layer 0
The 6 buses

Layer 0 is 6 buses. Apps plug in. Regulators verify.

Every agent, user, system, and event flows through the same 6 buses. Identity on entry. Attestation on every action. Policy at runtime. Payment pre-checked. Audit immutably recorded. Council as final arbiter.

🆔 Identity Bus

Every agent, user, and system gets a did:csoai — a W3C-compliant decentralized identifier. No vendor lock-in, no SAML cascade.

  • W3C DID v1.1 standard
  • IETF AIP alignment
  • Ed25519 + Secp256k1 key support
  • Works offline, works air-gapped

📜 Attestation Bus

Every compliance event becomes a signed, publicly-verifiable receipt. Your auditor, regulator, or customer can check the truth — without calling us.

  • Ed25519 signature per event
  • Public verify URL (csoai.org/verify/{id})
  • Solana SBT mint for material events
  • Sub-5ms verification latency

⚖️ Policy Bus

PDCA (Plan-Do-Check-Act) runtime enforces rules across all connected apps. Sub-0.1ms latency. 13-framework crosswalk in YAML.

  • 13 frameworks: EU AI Act, NIST AI RMF, ISO 42001, GDPR, DORA, NIS2, CRA, MDR, SR 11-7, BCBS 239, SOC 2, OCSF, custom
  • Runtime enforcement at agent-call latency
  • Versioned policies with Ed25519 signatures
  • Override workflow for emergency exceptions

💳 Payment Bus

x402 / AP2 / ACP payment flows pre-checked for compliance. The agent pays — and the receipt is cryptographic proof that the payment complied.

  • x402 micropayments with compliance attestation
  • AP2 agent payment protocol support
  • ACP agent commerce standard
  • Each transaction emits a Watchdog Certificate

📚 Audit Bus

Immutable, regulator-checkable logs. Append-only, hash-chained, optionally blockchain-anchored to Polygon PoA or Solana.

  • SHA-256 hash chain (every entry references prior)
  • Polygon PoA anchoring (optional, for regulators)
  • Solana SBT for material attestations
  • Indefinite retention (or your policy)

👥 Council Bus

BFT multi-stakeholder governance as a first-class system bus. Override any decision with quorum. Resolve disputes with cryptographic voting.

  • Byzantine Fault Tolerant (33 voters, f=10)
  • Weighted by stakeholder type (regulator/operator/customer)
  • Sub-5ms consensus latency
  • Audit trail per vote (Ed25519 signed)
