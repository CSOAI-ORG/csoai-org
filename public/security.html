How CSOAI is built for security & trust

Every Watchdog Certificate we issue is cryptographically signed, publicly verifiable, and built to be vendor-independent. Here's exactly how.

1. Ed25519 signatures, not opaque seals

Every Watchdog Certificate is signed with the Ed25519 algorithm — a high-performance elliptic curve signature standardised by the IETF (RFC 8032). Each certificate is a JSON document with:

  • ts — Unix timestamp of issuance
  • line — the canonical SIGIL line (immutable)
  • gloss — plain-English interpretation
  • digest — SHA-256 hash of the parsed line
  • prev_sig — previous record's signature (chain)
  • signature — Ed25519 signature of (digest + prev_sig)

You can verify any certificate yourself with meok-attestation-api.vercel.app/verify/{cert_id} — no login, no contact with us, no third party. The signature math is the trust.

Why Ed25519: NIST-recommended, ~10,000x faster than RSA-2048, no SHA-1 dependencies, 64-byte signatures, 32-byte keys. Used by SSH, TLS 1.3, Signal, and the entire modern cryptography stack.

2. Hash-chained ledger, not a database

Every certificate references the signature of the one before it. This means:

  • Tamper-evident: changing any historical record invalidates all subsequent signatures
  • Auditable: anyone can recompute the chain from the public ledger
  • Reproducible: the same input always produces the same chain

The SOV3 substrate maintains this ledger in a tamper-evident JSONL file at ~/clawd/sovereign-temple/data/sigil_ledger.jsonl. Every CSOAI Watchdog Certificate is also a sigil, so the chain covers both.

3. Public verify URLs, no lock-in

Every Watchdog Certificate has a public verify URL of the form meok-attestation-api.vercel.app/verify/{cert_id}. The endpoint:

  • Returns the certificate JSON
  • Verifies the Ed25519 signature cryptographically (no DB lookup needed)
  • Returns the chain position (which record in the ledger)
  • Is hosted on Vercel Edge (multi-region, sub-100ms response)

If CSOAI disappears, your certificates still verify. The Ed25519 key is published in the cert itself — anyone can verify with nacl or openssl.
